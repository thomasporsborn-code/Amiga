# My Amiga Games

Static GitHub Pages site for an Amiga game collection.

## Publish

1. Make sure `admin-data.json` contains the admin changes you want to publish.
2. Push this folder to a GitHub repository.
3. Enable GitHub Pages from the repository root branch.

## Important

`admin-data.json` is the file-based admin layer for custom games, overrides, and accepted data-quality findings.

## Value Research Flow

The app can also read external valuation research from `market-values.json`.
That file is meant to be generated by a separate valuation agent, while the site keeps ownership of review and final confirmation on the `Value Review` page.

1. Build the research template:
   ```bash
   node build-value-research-template.js
   ```

### Direct scrape mode

2. Fetch sold-item comps:
   ```bash
   node fetch-ebay-comps.js
   ```
3. Import suggested values back into `games.json`:
   ```bash
   node import-value-suggestions.js value-research-ebay.json
   ```

### Browser-assisted mode

Use this when eBay UK starts returning browser-challenge pages to the script.

2. Build a small capture plan:
   ```bash
   node build-ebay-browser-capture-plan.js --count=5
   ```
3. Open the URLs from `ebay-browser-capture-plan.json` in your normal browser.
4. Save each sold-results page as HTML into `ebay-browser-captures/` using the suggested `captureFile` names.
5. Import the saved HTML:
   ```bash
   node import-ebay-browser-captures.js
   ```
6. Convert the imported comps into review suggestions:
   ```bash
   node import-value-suggestions.js value-research-ebay.json
   ```

### Long-running local batches

To let valuation research run in small local passes:

```bash
node run-ebay-value-batches.js --batch-size=5 --max-rounds=12 --cooldown-ms=300000 --batch-delay-ms=2500
```
